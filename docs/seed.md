## 1. 项目愿景 (Vision)

打造一个**“美食底层逻辑库”**。不同于传统菜谱网站教“怎么做 (How)”，本项目致力于解释“为什么 (Why)”。
我们将利用物理、化学、生物学原理，对烹饪过程进行**解构、量化和重构**，为理科思维者和追求极致的烹饪爱好者提供一套可复用、可调试的烹饪解决方案。

* **Slogan:** 拒绝玄学，烹饪是一门精确的科学。
* **核心理念:** 像写代码一样做菜 (Cooking as Coding)。

## 3. 核心内容库：底层道理 (Core Logic Base)

### 3.1 典型案例解析 (MVP Content)

这是网站初期的核心引流内容，用科学解释传统经验。

#### **Case 01: 脆嫩芹菜的秘密 (渗透压)**

* **现象：** 芹菜直接炒容易软塌，口感不佳。
* **解决方案：** 炒前用少许油+盐腌制 10 分钟，洗净沥干后再炒。
* **底层逻辑 (Physics)：**
* **盐 (NaCl)：** 制造高渗透压环境，迫使植物细胞失水，细胞壁收缩致密，提升脆度。
* **油 (Lipids)：** 形成疏水保护层，减少炒制时的瞬间水分流失，防止氧化变色。



#### **Case 02: 肉类“上浆”原理 (水合作用与热障)**

* **现象：** 肉片直接炒容易老、柴。
* **底层逻辑 (Chemistry)：**
* **淀粉 (Starch)：** 在 60°C+ 发生糊化反应，形成半透膜，锁住肉纤维内部水分。
* **蛋清 (Protein)：** 提供额外的蛋白质屏障，缓冲高温对肉纤维的直接冲击。



#### **Case 03: 炒青菜的“热锅快炒” (叶绿素保护)**

* **现象：** 小火慢炒导致青菜变黄、出水。
* **底层逻辑 (Biology/Chemistry)：**
* **脱镁叶绿素反应：** 长时间加热会让细胞液中的有机酸溢出，酸性环境下镁离子流失，叶绿素变为褐黄色的脱镁叶绿素。
* **策略：** 猛火（High Heat）快速破坏细胞结构但迅速终止酶活性，在酸性物质大量破坏色素前出锅。



#### **Case 04: 奶白鱼汤的形成 (脂肪乳化)**

* **现象：** 大火煮出奶白汤，小火煮出清汤。
* **底层逻辑 (Physical Chemistry)：**
* **乳化 (Emulsification)：** 剧烈沸腾（机械力）将脂肪打散成微米级油滴。
* **胶体：** 溶出的明胶蛋白包裹油滴，形成稳定的水包油 (O/W) 乳液，散射光线呈现白色。

## 4. 网站架构设计 (Architecture)

网站将采用**结构化数据**展示，而非传统的散文式菜谱。

### 4.1 模块划分

| 模块 ID | 模块名称 | 功能描述 |
| --- | --- | --- |
| **MOD_01** | **原理层 (The Kernel)** | 解释美拉德反应、焦糖化、乳化、变性等核心概念。 |
| **MOD_02** | **参数层 (Variables)** | 食材的理化性质数据库（如：烟点表、含水量表、纤维密度）。 |
| **MOD_03** | **算法层 (Algorithms)** | 具体菜谱的 SOP (标准作业程序)，精确到克、秒、摄氏度。 |
| **MOD_04** | **调试区 (Debugging)** | 常见烹饪失败案例分析 (Root Cause Analysis)。 |

### 4.2 特色功能构想

* **单位换算器：** 将“少许”、“适量”一键转换为基于主材重量的百分比（如：盐量 = 肉重的 1%）。
* **热力图谱：** 可视化展示不同烹饪方法的温度曲线。
