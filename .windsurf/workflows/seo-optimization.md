# SEO 优化工作流

这个工作流会系统性地优化 SciCook 网站的 SEO 表现。

## 步骤 1: 元数据检查

检查所有页面的元数据完整性：
- 使用 `grep_search` 查找缺少 title 或 description 的页面
- 检查 Open Graph 标签是否完整
- 验证 Twitter Card 设置

## 步骤 2: 结构化数据验证

检查 JSON-LD 结构化数据：
1. 菜谱页面的 Recipe 结构化数据
2. 文章页面的 Article 结构化数据
3. 首页的 Website 结构化数据

使用 `read_file` 检查每个页面的结构化数据格式。

## 步骤 3: 内部链接优化

分析内部链接结构：
1. 检查是否有死链（404）
2. 验证所有内部链接的正确性
3. 优化锚文本描述

## 步骤 4: 图片 SEO 检查

检查所有图片的 SEO 优化：
1. 验证 alt 属性是否完整
2. 检查图片文件名是否合理
3. 优化图片大小和格式

## 步骤 5: 性能优化建议

检查页面性能：
1. 分析页面加载速度
2. 检查资源压缩情况
3. 验证缓存策略

## 步骤 6: 生成 SEO 报告

生成详细的 SEO 优化报告，包括：
- 发现的问题
- 优化建议
- 优先级排序

## 自动化修复

对于可以自动修复的问题：
1. 自动添加缺失的 meta 标签
2. 修正错误的 Open Graph 数据
3. 优化图片 alt 属性

## 输出示例

```markdown
# SEO 优化报告

## 发现的问题
### 高优先级
- [ ] 3 个页面缺少 description
- [ ] 5 张图片缺少 alt 属性

### 中优先级  
- [ ] 部分内部链接需要优化锚文本
- [ ] 结构化数据格式需要统一

### 低优先级
- [ ] 图片文件名可以更 SEO 友好

## 自动修复结果
✓ 已为 3 个页面添加 description
✓ 已为 5 张图片添加 alt 属性

## 手动修复建议
1. 优化内部链接的锚文本
2. 统一结构化数据格式
```
